## V8.1.7 (2026-02-24) - Hotfix: algo-update confirm flow

### ✅ 원인 분석 (확정)
- `algo-update` confirm 단계에서 `y` 입력 후 즉시 종료되는 문제는 `input_confirm()` 내부 변수 스코프 충돌로 확인되었습니다.
- 호출부가 `input_confirm answer ...`를 사용하고, 함수 내부에서도 동일 이름 변수를 사용할 경우 호출자 변수 반영이 누락될 수 있습니다.

### ✅ 수정 내용
- `input_confirm()`의 입력 임시 변수명을 분리하여 호출자 변수(`answer`)가 정확히 갱신되도록 수정했습니다.
- `y/n` 입력 결과가 호출자에 정상 전달되어 `algo-update`가 즉시 종료되지 않고 업데이트 로직으로 진입합니다.

### ✅ 회귀 방지
- `tests/test_input_confirm_scope.sh`를 추가해 비대화형 경로에서 `input_confirm`의 호출자 변수 반영을 검증합니다.
- `tests/run_tests.sh`, `tests/run_tests.ps1`에 신규 테스트 스위트를 포함했습니다.
